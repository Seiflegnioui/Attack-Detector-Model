# 🛡️ Network Intrusion Detection System (IDS) — NSL-KDD

Real-time network intrusion detection using supervised ML (XGBoost + Random Forest) on the [NSL-KDD](https://www.kaggle.com/datasets/hassan06/nslkdd) benchmark dataset. Inspired by **NVIDIA Morpheus**.

---

## 📋 Problem & Approach

Classic signature-based IDS fail against novel attacks. This project trains an intelligent IDS to classify network connections into 5 categories:

| Class | Description | Train Samples | Share |
|-------|-------------|:---:|:---:|
| **Normal** | Legitimate traffic | 67,343 | 53.5% |
| **DoS** | Denial of Service | 45,927 | 36.5% |
| **Probe** | Scanning / Reconnaissance | 11,656 | 9.3% |
| **R2L** | Remote to Local | 995 | 0.8% |
| **U2R** | User to Root | 52 | 0.04% |

Three pillars tackle this:
1. **SMOTE** inside the cross-validation pipeline — zero data leakage, handles extreme class imbalance
2. **RandomizedSearchCV + StratifiedKFold** — automated hyperparameter tuning
3. **FastAPI + Nginx** — real-time inference serving with an interactive dashboard

## 🔬 ML Pipeline

```
data/KDDTrain+.txt ──┐
data/KDDTest+.txt  ──┤
                     ▼
             prepare_data.py
    ┌─────────────────────────────────┐
    │  1. Load raw CSVs               │
    │  2. Map 39 labels → 5 classes   │
    │  3. One-hot encode categoricals  │
    │  4. Align test cols to train     │
    │  5. Save to data/processed/     │
    └────────────────┬────────────────┘
                     ▼
              train_model.py
    ┌─────────────────────────────────┐
    │  1. Fit StandardScaler (train)  │
    │  2. ImbPipeline:                │
    │       SMOTE → RF / XGBoost      │
    │  3. RandomizedSearchCV          │
    │       StratifiedKFold (3 folds) │
    │       F1 Macro scoring          │
    │  4. Evaluate on holdout test    │
    │  5. Benchmark latency           │
    │  6. Save best_model.pkl         │
    │  7. Save reports/*.png          │
    └────────────────┬────────────────┘
                     ▼
               api/api.py
         Loads best_model.pkl
         POST /predict        ← custom JSON input
         GET  /predict/random ← dashboard simulation
```

### Why SMOTE inside the pipeline?
SMOTE is applied **only on the training fold** during cross-validation — never on the validation fold. This ensures synthetic samples don't influence evaluation metrics, eliminating data leakage entirely. This is the correct approach; using `class_weight` is simpler but less powerful for extreme imbalance (U2R has only 0.04% of samples).

| Metric | Random Forest | **XGBoost** |

### Rare attack detection (the real challenge)
| Class | Precision | Recall | F1 |
|-------|:---:|:---:|:---:|
| U2R (52 train samples) | 71.88% | 34.38% | **0.46** |
| R2L | 97.11% | 21.45% | 0.35 |
| DoS | 96.57% | 97.97% | 0.97 |

### Inference latency
| Batch size | Time |
|---|---|
| 1 connection | ~7 ms |
| 1,000 connections | ~28 ms |
| 10,000 connections | ~218 ms |

> ~45,000 connections classified per second on a standard CPU.

## 🚀 Quickstart

### Step 1 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 2 — Prepare data
```bash
python src/prepare_data.py
```
Reads `data/KDDTrain+.txt` and `data/KDDTest+.txt`, saves processed splits to `data/processed/`.

### Step 3 — Train models
```bash
python src/train_model.py
```
Trains RF and XGBoost with SMOTE + CV, saves all `.pkl` files to `models/` and charts to `reports/`.

### Step 4 — Start the API
```bash
uvicorn api.api:app --reload
```
Swagger UI available at **http://127.0.0.1:8000/docs**

### Step 5 — Open the dashboard
Open `frontend/index.html` in your browser and click **"Démarrer la Simulation"**.
